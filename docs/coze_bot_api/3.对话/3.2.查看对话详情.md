查看对话的详细信息。 

在非流式会话场景中，调用发起对话接口后，可以先轮询此API确认本轮对话已结束（status=completed），再调用接口查看对话消息详情查看本轮对话的模型回复。 

说明
- 仅在对话开启了保存历史记录（auto save history=true）后，可通过此接口查看对话的详细信息。 

- 建议一个对话每秒轮询一次。

## 错误码

该接口有两处错误需要判断。一个位于回包的 .code ，对应该接口本身的错误；另一个位于回包的 .last_error.code ，对应/v3/chat的错误。
如果成功调用扣子编程的 API，返回信息中 code 字段为 0。如果状态码为其他值，则表示接口调用失败。此时 msg 字段中包含详细错误信息，但可以参考错误级文档查看对应的解决方法。
