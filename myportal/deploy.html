本地部署指南

把 ClassNode
装进你的电脑

无需云服务器。无论你习惯用安装包还是手动源码部署，都能在几分钟内完成。

方式一：安装包 方式二：源码部署 启动后访问
推荐 · 零门槛

方式一安装包安装（推荐）

从项目 Release 页面 或 下载渠道 下载对应系统的安装包，双击即可完成安装。这是最简单、最推荐的方式。

💡 提示：统信 UOS 等 Linux 系统没有现成的安装包，请下载源码包，然后按下方「源码部署」中的 PM2 一键安装方式完成部署（sudo bash install.sh）。

安装完成了？ 跳转到下方「启动后访问」，打开浏览器即可开始使用。
进阶 · 技术推荐

方式二本地源码部署

如果你有一定的技术基础，也可以下载源码通过命令行自行启动。Linux 平台建议使用此方式。Windows 和 macOS 用户建议直接使用安装包，体验最完整。

第一步：安装 Node.js

以统信 UOS 系统（ARM 架构）为例：

终端
# 1. 创建并进入软件目录
mkdir -p ~/software && cd ~/software

# 2. 下载（ARM 架构用 arm64，x86 架构用 x64）
wget https://mirrors.huaweicloud.com/nodejs/v24.16.0/node-v24.16.0-linux-arm64.tar.xz

# 3. 解压
tar -xvf node-v24.16.0-linux-arm64.tar.xz
mv node-v24.16.0-linux-arm64 nodejs24

# 4. 将 Node.js 加入环境变量
echo 'export PATH=$HOME/software/nodejs24/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

第二步：更换 npm 源（国内用户推荐）

终端
npm config set registry https://registry.npmmirror.com

# 检查是否切换成功
npm config get registry
# 预期输出: https://registry.npmmirror.com/

第三步：部署 ClassNode

从 Release 页面 或 下载渠道 下载源码压缩包（如 classnode-v1.x.x.tar.gz 或 .zip），解压后进入目录。

🐧 Linux 推荐方式（PM2 进程管理）：

终端
# 解压
tar -xzf classnode-v1.x.x.tar.gz
cd classnode-v1.x.x

# 一键安装（默认安装到 /opt/classnode）
sudo bash install.sh

安装脚本会自动完成：检测环境 → 安装依赖 → 构建项目 → 初始化数据库 → 配置 PM2 开机自启。
版本升级时同样运行 sudo bash install.sh，脚本自动备份和恢复数据。

管理命令：

终端
pm2 status              # 查看运行状态
pm2 logs classnode      # 查看日志
pm2 restart classnode   # 重启服务
pm2 stop classnode      # 停止服务

💡 也可以手动启动（不依赖 PM2）：

终端
# 进入项目目录
cd classnode-v1.x.x

# 安装依赖并构建
npm install -g pnpm
pnpm install
pnpm build
pnpm build:server

# 初始化数据库并启动
cd server && npx prisma db push && cd ..
node start.js
💡 提示：首次启动会自动安装依赖、初始化数据库并构建（约 1–5 分钟，需联网）。版本升级后首次启动也会因数据库更新而略有延迟，属正常现象。
部署完成了？ 跳转到下方「启动后访问」，打开浏览器即可开始使用。
两种方式通用

通用启动后访问

无论选择安装包还是源码部署，启动完成后，在浏览器中访问以下地址即可使用：

  • 管理端：管理电脑 打开浏览器访问 http://localhost:3001/teacher
  • 参与端：局域网内任意设备（平板、手机、其他电脑）访问 http://管理电脑IP:3001/classroom
💡 提示：参与端通过局域网访问，只需将 localhost 替换为管理电脑的局域网 IP 地址即可（如 http://192.168.1.100:3001/classroom）。
参与端无需安装任何软件，也无需互联网连接，打开浏览器就能用。
💡 客户端连不上？Windows 系统默认的安全策略有时会误拦截网络。若客户端连接失败，多半是管理机的 3001 端口被防火墙阻挡。请在管理电脑上尝试：
方式一：打开「Windows 安全中心 → 防火墙和网络保护 → 允许应用通过防火墙」，勾选 ClassNode（或 Node.js）的专用与公用网络（若无，点「允许其他应用」手动添加）。
方式二：进同页面的「高级设置 → 入站规则 → 新建规则」，按提示勾选：端口 → TCP → 输入 3001 → 允许连接。
