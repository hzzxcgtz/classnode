部署指南
把 ClassNode 装进你的电脑

无需云服务器，无论你习惯用安装包还是手动源码部署，都能在几分钟内完成。

方式一：安装包 方式二：源码部署 启动后访问
推荐 · 零门槛

方式一 安装包安装（推荐）

从项目 Release 页面 或 网盘渠道下载对应系统的安装包，双击即可完成安装。最简单、最推荐的方式。

💡 提示：统信 UOS 等 Linux 系统没有现成的安装包，请下载源码包，然后按下方「源码部署」说明完成安装部署。

安装包版本列表

平台 安装包
macOS Apple Silicon ClassNode_1.x.x_macos_apple-silicon.dmg
macOS Intel ClassNode_1.x.x_macos_intel.dmg
Windows（64 位） ClassNode_1.x.x_windows_x64-setup.exe
Windows（ARM64） ClassNode_1.x.x_windows_arm64-setup.exe
Linux classnode-v1.4.8x.x.zip（源码包）
💡 提示：安装或更新版本后首次启动时，程序会自动同步数据库结构，可能会有 10-30 秒 的短暂延迟，请耐心等待，后续启动将恢复正常速度。
安装完成了？ → 跳转到下方「启动后访问」部分，打开浏览器即可开始使用。
进阶 · 技术推荐

方式二 本地源码部署

如果你有一定的技术基础，也可以下载源码通过命令行自行启动。Linux 平台建议使用此方式。Windows 和 macOS 用户建议直接使用安装包，体验最完整。

1

第一步：安装 Node.js

以统信 UOS 系统（ARM 架构）为例：

终端
# 1. 创建并进入软件目录
mkdir -p ~/software && cd ~/software

# 2. 下载（ARM 架构用 arm64，x86 架构用 x64）
wget https://mirrors.huaweicloud.com/nodejs/v24.16.0/node-v24.16.0-linux-arm64.tar.xz

# 3. 解压
tar -xvf node-v24.16.0-linux-arm64.tar.xz
mv node-v24.16.0-linux-arm64 nodejs24

# 4. 将 Node.js 加入环境变量
echo 'export PATH=$HOME/software/nodejs24/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
2

第二步：更换 npm 源（国内用户推荐）

终端
npm config set registry https://registry.npmmirror.com

# 检查是否切换成功
npm config get registry
# 预期输出: https://registry.npmmirror.com/
3

第三步：部署 ClassNode

从 Release 页面 或 网盘渠道 下载源码压缩包（如 classnode-v1.4.8x.x.zip），解压后打开终端进入目录，运行启动脚本：

终端
# 进入项目目录（请以实际解压路径为准）
cd classnode-v1.4.8x.x

推荐方式：使用 Node.js 直接运行启动脚本

终端
node start.js

也可以在桌面双击 start-classnode-linux.sh，以终端方式运行（效果相同）。

💡 提示：首次启动会自动安装依赖、初始化数据库并构建（约 1-5 分钟，需联网）。后续启动时若数据库结构未变化则会跳过初始化，速度明显加快。版本升级后首次启动也会因数据库更新而略有延迟，属正常现象。
部署完成了？ → 跳转到下方「启动后访问」部分，打开浏览器即可开始使用。
两种方式通用

通用 启动后访问

无论你选择安装包还是源码部署，启动完成后，在浏览器中访问以下地址即可使用：

  • 教师端：教师电脑 打开浏览器访问 http://localhost:3001/teacher
  • 学生端：局域网内任意设备（平板、手机、学生机）访问 http://教师电脑IP:3001/classroom
💡 提示：学生端通过局域网访问，只需将 localhost 替换为教师电脑的局域网 IP 地址即可（如 http://192.168.1.100:3001/classroom）。
学生端无需安装任何软件，也无需互联网连接，打开浏览器就能用。
💡 贴心提示：学生机连不上？Windows 系统默认的安全策略有时会误拦截网络。若学生机连接失败，多半是教师机的 3001 端口被防火墙阻挡。请在教师电脑上尝试：
方式一：打开「Windows 安全中心 → 防火墙和网络保护 → 允许应用通过防火墙」，勾选 ClassNode (或 Node.js) 的专用与公用网络（若无，点「允许其他应用」手动添加）。
方式二：进同页面的「高级设置 → 入站规则 → 新建规则」，按提示勾选：端口 → TCP → 输入 3001 → 允许连接。
